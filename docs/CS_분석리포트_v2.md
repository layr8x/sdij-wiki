> 분석 일자: 2026-05-12

## 1. 왜 이 분석인가

AMS 나무 위키 로드맵 **1단계 — "CS 데이터 수집 및 지식 구조 설계"** 의 전제 조건은 *어떤 문의가 얼마나, 왜 반복되는지*를 정량적으로 파악하는 것이다. 이것이 위키 콘텐츠의 우선순위와 구조를 결정한다.

**마이클래스 카카오톡 상담 채널(고객 대면)** 과 **단과 상담실장 오픈채팅(내부 운영)** 은 동일한 시스템 이슈의 양면이다. 학부모가 "학원등록 연동이 안 된다"고 상담하면, 같은 시간 상담실장이 내부 채팅에서 "이관 부탁드려요"를 올린다. 두 채널을 교차 분석하면 **고객이 묻는 것 = 운영자가 답해야 하는 것 = 위키에 들어갈 핵심 항목**이라는 등식이 성립한다.

## 4. AMS 위키 콘텐츠 우선순위

**우선순위 공식**: 채팅 빈도 × 부정감정 비율 × 응답시간 편차 × 직원 채널 일치도

| 순위 | 위키 섹션 | 근거 데이터 | 콘텐츠 소스 |
|---|---|---|---|
| **P0** | **회원관리 > 통합회원/계정이관** | 고객 566건(40%) + 직원 33건(23.4%) | 단과 ↔ 플서실 이관 요청 패턴, 표준 처리 응답 |
| **P0** | **영상/VOD > 복습영상·재생 트러블슈팅** | 고객 440건(31%) + 부정감정 37% | 시스템 폴백 절차, 사용자 우회 가이드 |
| **P1** | **출결관리 > QR 출석·출석부** | 고객 60건 + 직원 18건(12.8%, 부정 38.9%) | QR 인식 실패 폴백, 출석부 인쇄 가이드 |
| **P1** | **결제/환불 SOP** | 고객 92건, 평균 응답 422분 | 중복결제·전환결제 케이스, 권한 분기 의사결정 트리 |
| **P1** | **인증·로그인 가이드** | 고객 32건, 부정 **46.9%** (최고) | 비밀번호 재설정·이메일 인증 표준 응답 |
| P2 | 강좌 신청/수강 | 고객 134건, 부정 17.9% | 정형화된 안내 |
| P2 | 학부모앱 / 자녀 추가 | 고객 189건(학부모/계정통합 포함) | 학부모앱 별도 채널 안내 + 자녀 ID 연결 |
| P3 | 교재/배송 | 고객 3건 (소량) | 배송 추적 |

## 10. 직전까지의 잘못된 보고 정정 기록

이 분석은 두 번 잘못 보고했음. 정정 트레일:

| 시점 | 보고 | 원인 |
|---|---|---|
| 1차 | "채팅방 100건 / 메시지 944건" | search API의 `size=100` cap을 한계로 오인. `has_next=true` 무시. |
| 2차 | "채팅방 100건 / 메시지 1,080건" | chatlogs 페이지네이션은 fix 했지만 채팅방 페이지네이션은 여전히 누락. |
| 3차 | "응답 P90 14.6시간 = 위기" / "차*희 87% SPOF" / "5월 peak" | 24h 평균 기반 SLA + 100건 표본 기반 매니저 분포. 도메인(운영시간·CS 부서 구조) 무시. |
