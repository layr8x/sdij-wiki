1 · 왜 이 분석인가

AMS 나무 위키 로드맵 1단계 — "CS 데이터 수집 및 지식 구조 설계" 의 전제 조건은 어떤 문의가 얼마나, 왜 반복되는지를 정량적으로 파악하는 것이다. 이것이 위키 콘텐츠의 우선순위와 구조를 결정한다.

마이클래스 카카오톡 상담 채널(고객 대면) 과 단과 상담실장 오픈채팅(내부 운영) 은 동일한 시스템 이슈의 양면이다. 학부모가 "학원등록 연동이 안 된다"고 상담하면, 같은 시간 상담실장이 내부 채팅에서 "이관 부탁드려요"를 올린다. 두 채널을 교차 분석하면 고객이 묻는 것 = 운영자가 답해야 하는 것 = 위키에 들어갈 핵심 항목이라는 등식이 성립한다.

7 · 위키 vs 프로덕트 구분

위키로 해결

  • 계정 이관·통합 — 권한 위임 + SOP 위키화
  • 환불·결제 SOP — 의사결정 트리로 422분 → 60분
  • 출석부 인쇄 가이드 — 기능 사용법 페이지
  • 수강신청 안내 — 부정 17.9%, 정형 응답

프로덕트로 고침

  • 영상 재생 오류 — 스트리밍 인프라 (440건 + 부정 37%)
  • 로그인·인증 UX — 부정 46.9% 최고
  • 학부모앱 자녀 추가 — 부정 41.8%
  • 카톡 채널 진입 카드 세분화 (89건 "마이클래스 앱 관련 문의")
  • 드롭 472건 회수 — 미응답 알림 자동화
